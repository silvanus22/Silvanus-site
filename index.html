name: Deploy Silvanus site on: push: branches: - main jobs: deploy: runs-on: ubuntu-latest steps: - name: Checkout repository uses: actions/checkout@v3 - name: Deploy to GitHub Pages uses: peaceiris/actions-gh-pages@v3 with: github_token: ${{ secrets.GITHUB_TOKEN }} publish_dir: ./ Пояснення: При пуші в гілку main запускається робота deploy. Репозиторій завантажується. Вміст поточної папки (./) публікується на гілку gh-pages. GitHub Pages можна налаштувати так, щоб брати контент із gh-pages. Що потрібно зробити, щоб це працювало: Створити файл .github/workflows/static.yml у репозиторії з цим вмістом. В налаштуваннях репозиторію у GitHub (Settings → Pages) вибрати джерело для GitHub Pages: гілку gh-pages і папку / (root). Кожен пуш у main автоматично публікуватиме останню версію сайту на GitHub Pages. Якщо хочеш, я можу: Згенерувати цей файл workflow і вставити в твій проєкт. Пояснити, як додати його до репозиторію.
